Specification

Each continuation eventually extends a primordial continuation. Each primordial continuation has a liquid environment, a set of bindings from liquid locations to values. The liquid environment of the primordial continuation of the current continuation is the current liquid store.

An identifier cannot only name a type of syntax or a location, but also a liquid location. An identifier that names a liquid is called a liquid variable or liquid, for short.

Syntax: ⟨liquid⟩

An expression consisting of a liquid is a liquid reference. The value of the liquid reference is the value in the current liquid environment for the liquid location named by liquid.

If the current liquid environment does not contain a binding for that liquid location, an exception with condition type &assertion is raised.

Syntax: set! ⟨liquid⟩

The ⟨expression⟩ is evaluated, and the current continuation replaced by an equivalent continuation except that the liquid environment of its primordial continuation is extended by binding the liquid location named by liquid to the value of expression. The result of the set! expression is unspecified.

If the current liquid environment does not contain a binding for that liquid location, an exception with condition type &assertion is raised.

Definition: define-liquid ⟨liquid⟩ [expression]

This definition binds ⟨liquid⟩ to a fresh liquid location before the expression is evaluated, and the current continuation replaced by an equivalent continuation except that the liquid environment of its primordial continuation is extended by binding the fresh liquid location to the value of expression.

the continuation of ⟨expression⟩ should not be invoked more than once. Implementations may or may not detect that the continuation of expression is invoked more than once. However, if the implementation detects this, it must raise an exception with condition type &assertion.

If the optional expression is not given, a side-effect-free expression returning an unspecified value is used instead.

Syntax: let-liquid ...

Syntax: liquid-let ...
